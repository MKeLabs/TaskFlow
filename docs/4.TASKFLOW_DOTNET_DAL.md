# Step 4: Data Access Layer (DAL)

> **Note:** The Project entity, repository, and CRUD logic are already covered in detail in Step 3. This step is optional and only needed if you want to see a minimal example of setting up a DbContext or additional entities.

## What & Why

The Data Access Layer (DAL) handles all interactions with the database. Separating data logic keeps your code organized and maintainable. If you already followed Step 3, you can skip this step or use it as a reference for adding more entities and configuring your DbContext.

### 1. DbContext Setup

In `TaskFlow.DAL/Context/TaskFlowDbContext.cs`:

```csharp
using Microsoft.EntityFrameworkCore;
using TaskFlow.DAL.Entities;

public class TaskFlowDbContext : DbContext
{
    public DbSet<ProjectEntity> Projects { get; set; }

    public TaskFlowDbContext(DbContextOptions<TaskFlowDbContext> options)
        : base(options) { }
}
```

### 2. Configure the Database Connection

To connect your `TaskFlowDbContext` to a database, you need to add a connection string to your `appsettings.json` and configure the DbContext in your API startup.

#### Add the Connection String

In `appsettings.json` (in the API project):

```json
"ConnectionStrings": {
  "TaskFlowDb": "Server=(localdb)\\mssqllocaldb;Database=TaskFlowDb;Trusted_Connection=True;MultipleActiveResultSets=true"
}
```

Adjust the connection string for your database provider (SQL Server, SQLite, PostgreSQL, etc).

A **connection string** tells your app how to connect to the database. It includes:
- **Server**: The address of your database server (e.g., `localhost`, `127.0.0.1`, or a remote server)
- **Database**: The name of the database to use
- **Authentication**: How to log in (e.g., `Trusted_Connection=True` for Windows auth, or `User Id=...;Password=...` for SQL auth)
- **Options**: Extra settings (e.g., `MultipleActiveResultSets=true` for SQL Server)

**Example for SQL Server LocalDB:**
```
Server=(localdb)\\mssqllocaldb;Database=TaskFlowDb;Trusted_Connection=True;MultipleActiveResultSets=true
```
- Use `Server=localhost;` for a full SQL Server instance.
- For SQLite: `Data Source=TaskFlowDb.db`
- For PostgreSQL: `Host=localhost;Database=TaskFlowDb;Username=postgres;Password=yourpassword`

> You can find more examples in the [official docs](https://www.connectionstrings.com/).

#### Configure the DbContext in Startup

In `Program.cs` (API project):

```csharp
using TaskFlow.DAL.Context;
using Microsoft.EntityFrameworkCore;

// ...

builder.Services.AddDbContext<TaskFlowDbContext>(options =>
    options.UseSqlServer(builder.Configuration.GetConnectionString("TaskFlowDb")));
```

- Use `UseSqlServer`, `UseSqlite`, or `UseNpgsql` depending on your database.
- This registers the DbContext so it can be injected and used throughout your app.

### 3. Add and Run Migrations

After creating your DbContext, entities, and configuring your connection string, you need to create and apply a migration to set up your database schema.

> **Note:**
> To use `Add-Migration` and `Update-Database` in the Package Manager Console, you need to have the **Microsoft.EntityFrameworkCore.Tools** package installed in your DAL project (`TaskFlow.DAL`).
> - Install it via NuGet Package Manager or run:
>   ```sh
>   dotnet add TaskFlow.DAL package Microsoft.EntityFrameworkCore.Tools
>   ```

**Recommended: Use the Package Manager Console in Visual Studio:**
- Open **Tools > NuGet Package Manager > Package Manager Console**
- Select the `TaskFlow.DAL` project as the Default Project (dropdown at the top)
- Run:
  - `Add-Migration InitialCreate`
  - `Update-Database`

**Alternative: Use the command line:**

From the root of your solution or the `TaskFlow.DAL` folder, run:

```sh
dotnet ef migrations add InitialCreate --project TaskFlow.DAL --startup-project TaskFlow.API
dotnet ef database update --project TaskFlow.DAL --startup-project TaskFlow.API
```

- The `--project` flag points to where your DbContext lives (usually DAL).
- The `--startup-project` flag points to your API (where configuration is loaded).

This will create the database and tables based on your models.
