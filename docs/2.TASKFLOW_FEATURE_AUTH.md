# Step 2: Authentication Feature

This step covers all code and setup for authentication: models, service, guard, and interceptor.

## What & Why

This step adds authentication to your app. You’ll create models, a service, a route guard, and an HTTP interceptor. These pieces work together to let users log in, protect routes, and automatically attach tokens to API requests. This is essential for any real-world app that needs user accounts and security.

## 1. Auth Models (`src/app/core/auth/auth.models.ts`)

## 2. Auth Service (`src/app/core/auth/auth.service.ts`)
```typescript
import { Injectable, computed, signal } from '@angular/core';
import { HttpClient } from '@angular/common/http';
import { catchError, of, tap } from 'rxjs';
import { LoginRequest, LoginResponse, MeResponse } from './auth.models';

const TOKEN_KEY = 'taskflow_access_token';

@Injectable({ providedIn: 'root' })
export class AuthService {
  private readonly _token = signal<string | null>(localStorage.getItem(TOKEN_KEY));
  readonly token = computed(() => this._token());
  readonly isAuthenticated = computed(() => !!this._token());

  constructor(private readonly http: HttpClient) {}

  login(request: LoginRequest) {
    return this.http.post<LoginResponse>('/api/auth/login', request).pipe(
      tap((res) => this.setToken(res.accessToken))
    );
  }

  register(request: LoginRequest) {
    return this.http.post<LoginResponse>('/api/auth/register', request).pipe(
      tap((res) => this.setToken(res.accessToken))
    );
  }

  logout() {
    this.clearToken();
  }

  me() {
    return this.http.get<MeResponse>('/api/auth/me').pipe(
      catchError(() => of({ id: null, email: null, roles: [] } satisfies MeResponse))
    );
  }

  private setToken(token: string) {
    localStorage.setItem(TOKEN_KEY, token);
    this._token.set(token);
  }

  private clearToken() {
    localStorage.removeItem(TOKEN_KEY);
    this._token.set(null);
  }
}
```

## 3. Auth Guard (`src/app/core/auth/auth.guard.ts`)
```typescript
import { CanActivateFn, Router } from '@angular/router';
import { inject } from '@angular/core';
import { AuthService } from './auth.service';

export const authGuard: CanActivateFn = (_route, state) => {
  const auth = inject(AuthService);
  const router = inject(Router);

  if (auth.isAuthenticated()) {
    return true;
  }

  return router.createUrlTree(['/login'], { queryParams: { returnUrl: state.url } });
};
```

**What is an Auth Guard?**

A guard in Angular is a function that decides if a route can be activated (navigated to). The **Auth Guard** checks if the user is logged in before allowing access to certain pages.

**Why use it?**
- Prevents unauthenticated users from accessing protected routes (like `/projects`).
- Redirects users to the login page if they are not authenticated.
- Improves security and user experience by controlling navigation based on login state.

**How will we use it?**
- We will *(don't do it yet)* add the guard to a route using `canActivate`:
  ```typescript
  { path: 'projects', component: ProjectsComponent, canActivate: [authGuard] }
  ```
- When a user tries to visit `/projects`, the guard runs. If not logged in, they are redirected to `/login`.


## 4. Auth Interceptor (`src/app/core/auth/auth.interceptor.ts`)
```typescript
import { HttpInterceptorFn, HttpRequest } from '@angular/common/http';
import { inject } from '@angular/core';
import { AuthService } from './auth.service';

export const authInterceptor: HttpInterceptorFn = (req, next) => {
  const auth = inject(AuthService);
  const token = auth.token();

  if (shouldSkipAuthorization(req.url) || !token) {
    return next(req);
  }

  return next(withAuthorizationHeader(req, token));
};

function shouldSkipAuthorization(url: string): boolean {
  return url.startsWith('/assets');
}

function withAuthorizationHeader(req: HttpRequest<unknown>, token: string): HttpRequest<unknown> {
  return req.clone({
    setHeaders: {
      Authorization: `Bearer ${token}`
    }
  });
}
```

**What are HTTP Interceptors?**

HTTP interceptors in Angular are special functions that can inspect and modify HTTP requests and responses. They act like **middlewares**: every outgoing request and incoming response passes through them. This lets you add, change, or log things globally, without repeating code in every service or component.

**Why are we using an Auth Interceptor?**

We use an auth interceptor to automatically attach the user’s JWT token to every API request that needs authentication. This way, you don’t have to manually add the token in every HTTP call—your app stays secure and your code stays clean.

**How does it work?**

- When your app makes an HTTP request, the interceptor checks if there’s a token and if the request should be authorized.
- If yes, it adds an `Authorization` header with the token to the request.
- If not (for example, requests to static assets), it leaves the request unchanged.
- The backend then uses this token to verify the user’s identity for protected endpoints.
