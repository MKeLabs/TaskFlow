# Step 6: Logging & Exception Middleware

## What & Why

Logging helps you track what happens in your application, while exception middleware catches and handles errors globally. This step shows how to set up both, using Serilog for logging and a custom middleware for error handling.

---

## 1. Logging Configuration (Serilog)

### What is Serilog? Why Seq?

**Serilog** is a popular logging library for .NET that makes it easy to write structured logs to many outputs (console, files, databases, etc.).

**Structured logging** means logs are more than just text—they have fields and properties, making them easier to search and analyze.

We are going to use **Seq** for local development. Its an easy-to-setup local logging server that's plug in play, that will let you view, search and analyze logs in a web UI. Its usually starting at the *http://localhost:5341* URL where you can access via the browser. 

### Required NuGet Packages

> **Where to install:**
> Add these packages to your API project (usually `TaskFlow.API`).

**Recommended: Use the NuGet Package Manager in Visual Studio:**
- Right-click the `TaskFlow.API` project in Solution Explorer
- Choose **Manage NuGet Packages...**
- Search for and install:
  - `Serilog.AspNetCore`
  - `Serilog.Sinks.Console`
  - `Serilog.Sinks.Seq`

**Alternative: Use the command line:**

```sh
dotnet add package Serilog.AspNetCore
dotnet add package Serilog.Sinks.Console
dotnet add package Serilog.Sinks.Seq
```

- You can add more sinks (outputs) as needed for your environment.

In `Program.cs`, configure Serilog:

```csharp
using Serilog;

builder.Host.UseSerilog((context, loggerConfiguration) =>
{
    loggerConfiguration
        .ReadFrom.Configuration(context.Configuration)
        .WriteTo.Console()
        .WriteTo.Seq("http://localhost:5341");
});
```

- This writes logs to the console and to a Seq server (optional).
- You can adjust the configuration in `appsettings.json` for more options.

## 2. Exception Middleware

`TaskFlow.API/Middleware/ExceptionMiddleware.cs`
```csharp
using System.Net;
using System.Text.Json;

namespace TaskFlow.API.Middleware;

public class ExceptionMiddleware(RequestDelegate next, ILogger<ExceptionMiddleware> logger)
{
    public async Task InvokeAsync(HttpContext context)
    {
        try
        {
            await next(context);
        }
        catch (Exception ex)
        {
            logger.LogError(ex, "Unhandled exception");
            context.Response.StatusCode = (int)HttpStatusCode.InternalServerError;
            context.Response.ContentType = "application/json";

            var payload = JsonSerializer.Serialize(new { message = "An unexpected error occurred." });
            await context.Response.WriteAsync(payload);
        }
    }
}
```

---

## 3. Registering the Middleware

In `Program.cs`, add the middleware to the pipeline:

```csharp
app.UseMiddleware<ExceptionMiddleware>();
```

Place this line before `app.UseAuthentication()` and `app.UseAuthorization()` to catch errors from all requests.
