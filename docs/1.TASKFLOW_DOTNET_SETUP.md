# Step 1: .NET Solution Setup

## What & Why

This step guides you through setting up the .NET backend for your TaskFlow application. You’ll create a solution, add projects, and configure the API for development. This is the foundation for your backend services.

## 1. Prerequisites
- [.NET 8 SDK](https://dotnet.microsoft.com/download)
- Visual Studio or VS Code
- (Optional) SQL Server or SQLite for local database

## 2. Create the Solution and Projects
Open a terminal and run:

```sh
dotnet new sln -n TaskFlow
cd TaskFlow

dotnet new webapi -n TaskFlow.API
```

You can add more projects (BLL, DAL) as needed:

```sh
dotnet new classlib -n TaskFlow.BLL
dotnet new classlib -n TaskFlow.DAL

dotnet sln add TaskFlow.API/TaskFlow.API.csproj
... # Add other projects
```

## 3. Run the API

From the TaskFlow.API folder:

```sh
dotnet run
```

The API will be available at `https://localhost:5001` (or similar).

---

## Visual Studio UI Alternative

If you prefer using Visual Studio’s graphical interface instead of the command line, follow these steps:

1. **Create a Solution and Projects**
   - Open Visual Studio.
   - Click **Create a new project**.
   - Choose **ASP.NET Core Web API** for your backend, name it `TaskFlow.API`, and click **Next**.
   - (Optional) Add **Class Library** projects for `TaskFlow.BLL` and `TaskFlow.DAL` the same way.
   - When prompted, create a new solution or add to an existing one.

1. **Set Startup Project**
   - Right-click `TaskFlow.API` and select **Set as Startup Project**.

1. **Run the API**
   - Press **F5** or click the green play button to build and run your API.
   - The API will launch and open in your browser.

**Tip:** Visual Studio handles most configuration for you, including launch settings and HTTPS certificates.
