# Step 1: Project Setup and Proxy

## What & Why

This step sets up your Angular project and configures a development proxy. The proxy lets your Angular app talk to your .NET API without CORS errors, making local development smooth and realistic. You’ll also learn why this is needed and how it works.

## 1. Prerequisites
- [Node.js](https://nodejs.org/) (LTS recommended)
- [Angular CLI](https://angular.dev/tools/cli) (`npm install -g @angular/cli`)
- .NET API running (e.g., at `https://localhost:7226`)

## 2. Create a New Angular Project
```bash
ng new taskflow-web --style=scss --routing --standalone
cd taskflow-web
```
Choose **SCSS** for styles and enable **routing** when prompted.

## 3. Set Up the Proxy for API Calls

When developing locally, your Angular app runs on `http://localhost:4200` and your API runs on `https://localhost:7226`. Browsers block requests from one origin to another (CORS). To avoid CORS issues and make development easier, you can set up a proxy so that API calls from Angular are transparently forwarded to your backend.

**How it works:**
- You call `/api/...` from Angular.
- The Angular dev server (ng serve) intercepts these calls and forwards them to your API.
- Your code never needs to know the real backend URL during development.

Create `proxy.conf.json` in the project root:
```json
{
  "/api": {
    "target": "https://localhost:7226",
    "secure": false,
    "changeOrigin": true,
    "logLevel": "debug"
  }
}
```

Update `package.json` to use the proxy when starting:
```json
{
  // ...existing code...
  "scripts": {
    // ...existing scripts...
    "start": "ng serve --proxy-config proxy.conf.json"
  }
}
```
