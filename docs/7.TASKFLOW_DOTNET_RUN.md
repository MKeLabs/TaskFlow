# Step 7: Running & Debugging the API

## What & Why

Running your API lets you test endpoints and see your backend in action. Visual Studio and the .NET CLI both offer ways to run and debug your project. Launch settings help you control how your app starts.

---

### 1. Run with Visual Studio (Recommended)

- Open the solution in Visual Studio.
- Right-click `TaskFlow.API` and select **Set as Startup Project**.
- Press **F5** (or click the green play button) to build and run the API.
- Visual Studio will open a browser window to your API’s launch URL.
- You can set breakpoints and debug your code as it runs.

**Tip:** Visual Studio uses `Properties/launchSettings.json` to control environment variables, launch URLs, and profiles. You can edit this file to:
- Change the default port
- Set environment variables
- Add custom launch profiles (e.g., for Docker)

---

### 2. (Second option) Run from the Command Line

From the `TaskFlow.API` folder:

```sh
dotnet run
```

- The API will be available at the URL shown in the console (e.g., `https://localhost:5001`).
- Use tools like [Postman](https://www.postman.com/) or [curl](https://curl.se/) to test your endpoints.

---

### 3. Debugging Tips

- Use breakpoints in Visual Studio to pause execution and inspect variables.
- The **Output** and **Debug** windows show logs and errors.
- You can attach the debugger to a running process if needed.
