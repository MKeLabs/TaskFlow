# Step 2: API Structure & Controllers

## What & Why

This step explains the structure of your .NET API project and how to add controllers for handling HTTP requests. Controllers are the entry points for your frontend to interact with the backend.

## 1. Project Structure

Typical folders:
- `Controllers/` — API endpoints
- `Services/` — Business logic
- `Infrastructure/` — Cross-cutting concerns (e.g., seeding, middleware)
- `Options/` — Configuration
- `Auth/` — Authentication helpers
- `Entities/` — Data models (in DAL)

## 2. Example Controller

Create a new file in `Controllers/`:

```csharp
using Microsoft.AspNetCore.Mvc;

[ApiController]
[Route("api/[controller]")]
public class ProjectsController : ControllerBase
{
    [HttpGet]
    public IActionResult GetAll()
    {
        // Return a list of projects (replace with real logic)
        return Ok(new[] { new { Id = 1, Name = "Demo Project" } });
    }
}
```

**What this does:**
- `[ApiController]` and `[Route]` set up routing for HTTP requests.
- The `GetAll` method responds to `GET /api/projects`.
